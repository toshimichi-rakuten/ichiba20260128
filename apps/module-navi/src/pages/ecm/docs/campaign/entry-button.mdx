# エントリーボタン

エントリーキャンペーンに使用する、エントリーするためのボタンパーツです。

<Space />

## 使い方

エントリーボタン用の javascript を追加の上、モジュールのソースコードをコピーし、キャンペーンコードを正しく記入して利用してください。  
モジュールは大きく分けて、Basic と Advance の２種類です。  
通常のエントリーボタンを使用する場合は、Basic を使用してください。  
通常と異なる使い方をする場合は、Advance より使用シーンに合わせて使ってください。  
選び方は、[詳細マニュアル](https://confluence.rakuten-it.com/confluence/pages/viewpage.action?pageId=3386152014) （閲覧は社内環境限定）をご確認ください。 ※特に Advance を使用する際

<Space />

＜種類＞

<Space />

**Basic**

- Default（SP）
- Default（PC）
- With Checkbox（SP）
- With Checkbox（PC）
- Default Checked Checkbox

**Advance**

- Matrix Pattern
- With Custom Button
- Custom Checkbox
- Linked Campaign Codes
- Dynamic Ekey

以下の javascript をアウトラインファイル下部の=== ページJavaScript ===部分へ追記して読み込ませてください。

## 注意事項

- コピーしたソースコードの ★★★ 部分を適宜編集してください。

  - 市場の企画にはキャンペーンコード冒頭に、/ic/ が付いているので、忘れない様ご注意ください。  
    　例：
    　キャンペーンページの URL が「`https://oubo.rakuten.co.jp/apply/ic/card20180505point5`」の場合、 ソースコードに記載するキャンペーンコードは、「`/ic/card20180505point5`」。

※キャンペーンコードの冒頭の / は、記載有無を問わずエントリーは機能します。

- 注意文言については、下記ルールを参照の上編集してください。（閲覧は社内環境限定）
  - [メールパーミッション取得文言の UI ルール](https://confluence.rakuten-it.com/confluence/pages/viewpage.action?pageId=1779542474)
  - [★RUP★ リワード系企画表記統一 Project - Reward campaign Unify notation Project](https://confluence.rakuten-it.com/confluence/pages/viewpage.action?pageId=3153249132)

※表示チェックの観点ではないため、表示管理責任者からの指摘はしませんが、必ず各担当者でのご確認お願いいたします。

#### Linked Campaign Codes

参照用キャンペーンコード（`linkedCodes`）のどれかに ユーザーがエントリー済みの場合、実施中のキャンペーンコード（`campaignCode`）のエントリー状況に関わらず、ボタンのステータスはエントリー済みとなります。  
「過去の同じ企画にエントリーしたことのあるユーザーは、今回のキャンペーンでエントリーできない」という様な設定をする際に使用してください。

<Space />

ソースコードをコピーし、 ★★★ 部分を、下記内容に沿って適宜編集してください。

<Space />

▽ 実施中キャンペーンコード

- エントリーさせたいキャンペーンコードを入れてください

▽ 参照用キャンペーンコード

- ユーザーが既にエントリーしているかどうかのチェックの際に使います。
- エントリー済みかどうかを確認したいキャンペーンコードを入れてください。
- 設定可能なキャンペーンコードは 20 個までです。21 個以上設定した場合、冒頭から 20 個のみが使われます。
- 半角カンマ「,」で区切って入力してください。

### Classes (advance)

With Custom Button を使用している場合は、ステータスに応じてこれらのクラスが適用されます。
