## 新モジュールへの移行に関して

**既存ページのモジュールインクルードパスを書き換えましたが、新モジュールが利用できません。**  
インクルードパスを書き換えるだけではなく、アウトラインファイルを新しくする必要があります。  
移行方法については、[新モジュールへの移行方法](/ecm/docs/how-to-replace/)ページをご確認ください。
<Space />

**新アウトラインファイルを利用していますが、アルコルが表示されません。**  
新アウトラインファイルでは、新モジュールに対応したアルコルのテンプレートを利用するか、モジュールを利用せず独自にCSS/JSを適用する必要があります。  
アルコル設定画面でテンプレート名を確認し、ecm_で始まるテンプレートを利用している場合は、新モジュール対応のテンプレートを利用したアルコルを新規に作成し、新モジュールのソースコードへ移行する必要があります。  
ecm_で始まらない独自テンプレート利用を利用していて、過去モジュールに依存している場合(レイアウトが表示されない場合)で、独自CSS/JSの対応が難しい場合は、[Design / Coding JIRA Forms](https://confluence.rakuten-it.com/confluence/pages/viewpage.action?pageId=1421125620)よりご相談ください。

<Space />

**新アウトラインファイルに差し替えたら、ページのJSが動かなくなりました。**  
新アウトラインファイル・モジュールパーツでは、HTML構造・CSSクラス名が変更されています。そのため旧モジュールHTML構造・CSSクラス名を利用したページ独自JSについては、修正が必要になります。モジュールのクラス名などの新旧対照は、[新モジュールへの移行方法](/ecm/docs/how-to-replace/)ページ掲載の[移行パーツ対照情報ツール](/ecm/docs/how-to-replace/#移行パーツ対照情報ツール)をご利用ください。  
JSの修正は原則各自制作元・制作ベンダーでの対応をお願いします。対応が難しい場合は、[Design / Coding JIRA Forms](https://confluence.rakuten-it.com/confluence/pages/viewpage.action?pageId=1421125620)よりご相談ください。

<Space />

**新旧モジュールを両方並行して読み込み使用してもよいですか**  
ページパフォーマンス低下、javascriptの重複読み込みなどによる不具合発生の原因になりますので、新旧モジュールの両方の読み込みは不可です。

<Space />

**今まで利用していたモジュールパーツ、CSSユーティリティが、新モジュールで見つかりません。**  
モジュールのクラス名などの新旧対照は、[新モジュールへの移行方法](/ecm/docs/how-to-replace/)ページ掲載の[移行パーツ対照情報ツール](/ecm/docs/how-to-replace/#移行パーツ対照情報ツール)をご利用ください。
利用率が少ないもの・推奨されないものに関しては、新モジュールへ移行されていないものがあります。代わりのCSSユーティリティを組み合わせて対応いただく、または、個別ページCSSで対応いただくようお願いいたします。

## 使い方に関して

**使いたいモジュールが見つかりません**  
Module Navi 左ナビゲーション上部に検索窓があり、フリーワード検索が可能です。  
原稿タイプモジュールについては、[AdFinder](/ecm/docs/ad-finder/)で絞り込み探すことができます。  
原稿タイプ以外のUIパーツについては、[UIパーツ一覧](/ecm/docs/ui/all/)ページで確認できます。

<Space />

**新アウトラインファイルを利用していますが、CSS/JSが効いていません。**  
新アウトラインファイルはダウンロード後、インクルードパスを各自グループの指定されたパスに編集する必要があります。  
[新モジュールへの移行方法](/ecm/docs/how-to-replace/)ページを確認してください。

<Space />

**SP/PCワンソース利用はできますか**  
SP用・PC用の表記がないパーツについては、ワンソース利用可能です。原稿タイプモジュールでワンソース利用をする場合は、枠（入稿・効果測定）がPC/SP同一である必要がありますのでご注意ください。

<Space />

**モジュールのデザインは変更可能ですか**  
はい、個別ページのCSSを追加いただくことでデザイン変更は可能です。  
ただし、その場合はモジュール運用側では検証対象外となりますので、各自で責任を持って検証してください。  
また、個別ページのCSSで指定する場合は、直接モジュールクラスに対して上書き指定するのではなく、ページ固有のクラス名と合わせて指定をすることを推奨します。[新モジュールへの移行方法 ＞ コーディング注意点](/ecm/docs/how-to-replace/#コーディング注意点)

<Space />

**モジュールのフォントカラーやフォントサイズ、マージンなどは指定可能ですか**  
原稿タイプモジュールや、テキストリンク色については、共通指定のフォントサイズなどが指定されていますので、変更せず利用することを推奨します。  
その他UIパーツモジュールについては、各自指定可能です。グリッドデザインに則った指定を行うことを推奨します。[CSSユーティリティ](/ecm/docs/css/)を利用いただけます。
