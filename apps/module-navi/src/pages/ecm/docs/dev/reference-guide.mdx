# リファレンスガイド

リファレンスは、**MDX**（Markdown上でJSXが使用出来る）をベースにして作成されております。  
そのため、リファレンスの更新作業には、Markdown記法の知識があることが推奨されます。

<Space />

<Note type='info'>
  [参考サイト](https://mdxjs.com/table-of-components/)
</Note>

## 作成ガイド

リファレンスの作成は、本リファレンスガイドと既存のリファレンスファイルの中身を参考に、作成を進めてください。

### リファレンスファイルの作業ディレクトリ

`apps/module-navi/src/pages/`配下になります。（2023/12の時点）  
該当のファイルを探して、修正してください。

#### 新規ファイルを作成するとき

1. ファイル名を決め、`*.mdx` ファイルを作成する

    <Space />

    `sample.mdx` のようにファイルを追加し、`apps/module-navi/src/pages/ecm/docs/〜` 配下の該当ディレクトリ先に格納します。

    <Space />

    例）

    > `apps/module-navi/src/pages/ecm/docs/ui/sample.mdx`

<Space />

2. 作成したファイルを読み込む

    <Space />

    `apps/module-navi/src/toc.ts` ファイルで先ほど作成したファイルを読み込みます。  
    読み込み方法はファイル内の他モジュールを参考にしてください。

    <Space />

    例）

    > `import { meta as Sample } from 'src/pages/ecm/docs/ui/sample.mdx'`  
    > 〜 省略 〜  
    > `export const toc = [*, Sample, *]`

### 必要なHTMLファイルの import

リファレンスで読み込んで利用したいHTMLファイルは、下記のように `import` してください。  
基本的に、`import` されたHTMLファイルは後述の「Code」「Html」コンポーネントで利用します。

<Space />

<Code content={`
import defaultHtml from 'ecm/src/modules/floating/default.html'
`} />

### アトリビュートの記述

アトリビュートは、機能ごとにタブで分けて記載してください。  
例えば、[アコーディオンメニュー](/ecm/docs/ui/accordion/#アトリビュート一覧)では、「アトリビュート」「トリガー アトリビュート」「コンテンツ アトリビュート」と3つのタブに分かれております。

<Space />

ただ、[上に戻る](/ecm/docs/ui/scroll-to-top/#アトリビュート一覧)のように複数に分ける必要がない場合、タブは使用せず直接テーブルコンポーネントを使用してください。

## コンポーネント（MDX）のチートシート

`*.mdx` ファイルの上部で下記を `import` することにより、リファレンス内でレイアウトのためのコンポーネントが利用できます。

- スペース（\<Space />）
- カード（\<Note type='*'>〜\</Note>）
- Code（\<Code />）
- Html（\<Html />）
- Blockquote（>）
- タブ（\<Tabs>〜\</Tabs>）
- テーブル（\<table>〜\</table>）

#### 使い方

- カードの種類は4種類（default, info, warn, error）あるため目的に応じて使い分けてください
- カード内にはリストは使用しないでください
- カードで使用される文章は最大2行に収めてください
- 基本的には左端のインデントは揃えてください（「書き方」を参考）

<Space />

<Note type='warn'>
  行数が3つ以上になるときや、文章が長くなるときには使用しないでください。（代替としてはカードのコンポーネントをやめて、Markdownの「リスト」などを使用してください。）
</Note>

<Space />

**良い例**

<Note type='default'>
  テキストテキストテキストテキストテキストテキスト
</Note>

<Space />
